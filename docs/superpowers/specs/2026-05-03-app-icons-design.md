# App Launcher Icons (Android + iOS)

**Date:** 2026-05-03

## Problem

Both Android and iOS still ship with the default Flutter launcher icon. Pre-MVP physical-device testing requires a recognizable app icon so testers can identify the app on the home screen, distinguish it from other Flutter apps installed for development, and surface obvious branding regressions in screenshots and bug reports.

Current state:

- `peppercheck_flutter/android/app/src/main/AndroidManifest.xml` references `@mipmap/ic_launcher`, and `mipmap-{m,h,xh,xxh,xxxh}dpi/ic_launcher.png` are the default Flutter "F" icon.
- `peppercheck_flutter/ios/Runner/Assets.xcassets/AppIcon.appiconset/` contains the default Flutter PNG set across the standard 15 size variants.
- A branded master image already exists at `store-listing/peppercheck_icon_512.png` (512×512, RGB, no alpha). It depicts a beige background, a white rounded square frame, a dark inner rounded square, and a chili-pepper glyph with a check mark.

## Design

### Source of truth

`store-listing/peppercheck_icon_512.png` is the single source of truth for the launcher icon. It is reused for store listings, marketing assets, and the launcher icon generator. The Flutter package does not keep its own copy.

### Tool

Use the `flutter_launcher_icons` package as the build-time generator:

- Added to `peppercheck_flutter/pubspec.yaml` under `dev_dependencies` (latest stable).
- Configured under a top-level `flutter_launcher_icons:` key in the same `pubspec.yaml`.
- Run once with `dart run flutter_launcher_icons` from `peppercheck_flutter/`. Generated PNGs and `Contents.json` are committed to git; the generator is not re-run on every build.

### Configuration

```yaml
flutter_launcher_icons:
  android: true
  ios: true
  image_path: "../store-listing/peppercheck_icon_512.png"
  remove_alpha_ios: true
```

Notes:

- `image_path` resolves relative to the `pubspec.yaml` directory (`peppercheck_flutter/`). The `..` prefix points back to the repo-root `store-listing/` folder so the master asset is referenced in place rather than duplicated.
- `remove_alpha_ios: true` is defensive. The current source PNG is already RGB without an alpha channel, but the App Store rejects icons that carry alpha; keeping the flag protects against a future master image saved as RGBA.
- `android: true` writes plain legacy mipmap PNGs across all density buckets and leaves the existing `@mipmap/ic_launcher` reference in `AndroidManifest.xml` untouched. No adaptive-icon foreground/background split is configured (see "Out of scope").

### Visual behavior

Per the user-confirmed approach, the master image is consumed as-is. The beige outer area of the source image is intentionally retained, so:

- On iOS, the system superellipse mask trims the four corners of the 512×512 square. The trimmed area is the beige border, so the inner white-rounded-square / dark-rounded-square / chili composition stays intact. A thin beige ring will remain visible between the iOS mask edge and the inner white frame.
- On Android pre-API 26 the full square image is shown.
- On Android API 26+ without an adaptive icon configuration the system falls back to legacy compatibility mode and renders the square inside its current home-launcher mask. The chili glyph is centered with sufficient padding that the launcher mask does not clip it, but the result is less crisp than a native adaptive icon.

These trade-offs are accepted for MVP physical-device testing. They are revisited before store submission (see "Out of scope").

### Manual verification

After running the generator and committing the result:

- `cd peppercheck_flutter && flutter build apk --debug -t lib/main_debug.dart` succeeds.
- Install the debug APK on the Android emulator (or a physical device); confirm the home-screen launcher shows the chili icon, not the default Flutter "F".
- Build and run on an iOS simulator (or a physical device); confirm the home-screen launcher shows the chili icon and that the system superellipse mask does not clip the chili glyph.
- Visually confirm the icon at the small notification / settings sizes on both platforms by opening Settings → Apps (Android) and the Spotlight search list (iOS).

### Out of scope

- Android adaptive icon (separate `adaptive_icon_foreground` and `adaptive_icon_background` layers).
- A 1024×1024 (or larger) master image. The current 512×512 source is upscaled by the generator for `Icon-App-1024x1024@1x.png`. Acceptable for MVP testing; needs a higher-resolution master before App Store submission.
- iOS dark and tinted icon variants (added in iOS 18). The generator emits only the single light variant.
- Splash screen / launch image refresh (`LaunchImage.imageset` is untouched).
- Renaming `store-listing/peppercheck_icon_512.png`, reorganizing the `store-listing/` directory, or moving the master into a dedicated branding folder.
